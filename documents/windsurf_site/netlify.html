Netlify Deployment

Automated Deployment for Modern Web Projects

Netlify provides a seamless platform for deploying and hosting modern web applications with continuous deployment, serverless functions, and more. This guide will help you deploy your VibeCode 101 projects quickly and easily.

Setting Up Your Netlify Account

1
Visit Netlify

Navigate to netlify.com in your web browser and click the "Sign up" button in the top right corner.

2
Sign Up for an Account
Available Sign-up Methods:
  • GitHub - Recommended - Seamless integration with your GitHub repositories
  • GitLab - Good option if you use GitLab for version control
  • Bitbucket - For teams using Atlassian's Bitbucket
  • Email - Basic signup with email and password
Pro Tip: We strongly recommend signing up with GitHub since this is what we'll use in this course. This allows for seamless continuous deployment directly from your GitHub repositories.
3
Authorize and Complete Setup
For GitHub Authentication:
  1. Click "Authorize Netlify" when prompted
  2. Choose whether to grant access to all repositories or select specific ones
  3. For this course, we recommend granting access to all repositories for simplicity
  4. Complete any additional account setup steps (team name, etc.)
For Email Authentication:
  1. Enter your full name and password
  2. Check your email inbox for a verification link
  3. Click the verification link to activate your account
  4. Complete the team setup process
Pro Tip: The free tier of Netlify is very generous and likely sufficient for all your VibeCode 101 projects. It includes unlimited personal projects, HTTPS, continuous deployment, and more.

Deploying Your First Site

1
Connect to Git Provider

Log in to Netlify and click "Add new site" > "Import an existing project" > "GitHub".

Authorize Netlify to access your GitHub repositories.

Choose the repository you want to deploy.

2
Configure Build Settings

For basic static sites, you can leave the build command empty and just specify your publish directory (e.g., . for the root folder).

For sites built with frameworks:

  • For a site without a build process: Leave the build command empty and set the publish directory to your site folder
  • For a site with a build process: Enter the build command (e.g., npm run build) and the output directory (e.g., build or dist)
3
Deploy Your Site

Click "Deploy site" and wait for the build and deployment process to complete.

Once deployed, Netlify will provide you with a unique URL (e.g., your-project-name.netlify.app).

4
Continuous Deployment

Netlify will automatically redeploy your site whenever you push changes to the connected branch in your GitHub repository.

Pro Tip: If your site requires environment variables (like API keys), you can set them in Netlify's dashboard under Site settings > Build & deploy > Environment variables.

Setting Up a Custom Domain

1
Add a Custom Domain

In your Netlify site dashboard, go to "Domain settings" > "Add custom domain".

Enter your domain name and click "Verify".

2
DNS Configuration Options

Netlify offers two options for DNS configuration:

  • Option 1: Use Netlify DNS (recommended) - Transfer your domain's nameservers to Netlify
  • Option 2: Use external DNS - Add CNAME or A records at your domain registrar
3
Using Netlify DNS

If you choose Netlify DNS, click "Set up Netlify DNS" and follow the instructions to update your domain's nameservers at your registrar.

Netlify will automatically configure everything, including SSL.

4
Using External DNS

If you prefer to use your current DNS provider:

  1. Add a CNAME record for www pointing to your Netlify site URL
  2. Add either an ALIAS, ANAME, or A record for the apex domain (e.g., example.com) pointing to Netlify's load balancer IPs
5
SSL/HTTPS Setup

Netlify provides free SSL certificates through Let's Encrypt.

After your DNS is configured, Netlify will automatically provision an SSL certificate for your domain.

Pro Tip: Using Netlify DNS simplifies the process and gives you access to additional features like branch subdomains. It's generally recommended unless you have specific needs for your current DNS provider.

Advanced Netlify Features

Netlify Functions

Build serverless functions to extend your static site with dynamic functionality. Functions are automatically deployed alongside your site.

# Create a netlify.toml file in your project root [functions] directory = "functions"

Branch Deploys & Preview URLs

Netlify can automatically deploy all branches of your repository, giving each a unique URL for previewing changes before merging.

Enable this in Site settings > Build & deploy > Continuous Deployment > Deploy contexts.

Netlify Forms

Collect form submissions without any server-side code by adding the netlify attribute to your HTML forms.

<form name="contact" method="POST" data-netlify="true"> <input type="text" name="name" /> <button type="submit">Send</button> </form>

Identity & Authentication

Add user authentication to your static site with Netlify Identity, supporting email/password, social logins, and more.

Enable this in Site settings > Identity > Enable Identity.

Pro Tip: For more complex sites, create a netlify.toml file in your repository root to version control your build settings, redirects, headers, and more.
