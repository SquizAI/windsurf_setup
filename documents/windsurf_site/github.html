GitHub Integration

Version Control for AI-Powered Development

GitHub provides a robust platform for version control, collaboration, and code hosting which is essential for modern AI-powered development workflows. This guide will walk you through setting up GitHub and integrating it with your VibeCode 101 projects.

Setting Up Your GitHub Account

1
Create a GitHub Account

Navigate to github.com in your web browser.

Sign Up Process:
  1. Enter your email - Use a professional email you check regularly
  2. Create a password - Must include at least 8 characters with a number and lowercase letter
  3. Choose a username - This will be visible in your project URLs, so choose wisely
  4. Opt in/out of email preferences
  5. Verify your account - Solve the security puzzle
  6. Click "Create account"
Pro Tip: Choose a username that aligns with your professional identity. Many developers use their GitHub profiles for job applications, so a professional username is recommended.
2
Verify Your Account

Solve the puzzle to verify you're human, then click "Create account".

Check your email for a verification link and click it to activate your account.

3
Set Up Two-Factor Authentication (Recommended)

For added security, go to Settings > Security > Two-factor authentication.

Follow the steps to set up 2FA using an authenticator app or SMS.

Pro Tip: Choose a username that reflects your professional identity, as it will be visible in your project URLs and contributions across GitHub.

Creating and Managing Repositories

1
Create a New Repository

Click the "+" icon in the top right corner and select "New repository".

Enter a repository name, description, and choose visibility (public or private).

Initialize with a README file to make cloning easier.

2
Clone the Repository Locally

On your repository page, click the green "Code" button and copy the HTTPS URL.

Open your terminal and run:

git clone https://github.com/yourusername/your-repo.git
3
Add Collaborators (Optional)

Go to Settings > Collaborators and add team members by username or email.

Choose appropriate permission levels for each collaborator.

Pro Tip: For VibeCode projects, create a clear README.md file that explains the purpose of your project, installation instructions, and how others can contribute.

Advanced GitHub Features

GitHub Actions

Automate your workflow with custom CI/CD pipelines directly in your repository.

GitHub Pages

Host your web projects directly from your repository with automatic deployment.

Security Features

Protect your code with dependency scanning, secret detection, and vulnerability alerts.

Project Management

Organize your work with project boards, issue tracking, and milestone planning.

5
Merge the Pull Request

After review, click "Merge pull request" to add your changes to the main branch.

Pro Tip: Write descriptive commit messages that explain why you made a change, not just what changed. This makes it easier for others (and your future self) to understand the purpose of each commit.

Integrating GitHub with Windsurf

1
Open Your Project in Windsurf

Launch Windsurf and open your project that's connected to a GitHub repository.

2
Use Git Controls in Windsurf

Windsurf provides built-in Git controls for common operations:

  • Click the branch icon to switch branches or create new ones
  • Use the source control tab to stage, commit, and push changes
  • Check file history and blame information directly in the editor
3
Leverage AI for Git Operations

Ask Cascade to help with Git operations by typing commands like:

  • "Create a commit message for my changes"
  • "Help me resolve this merge conflict"
  • "Create a .gitignore file for a React project"
4
Set Up GitHub Actions (Advanced)

Create automated workflows with GitHub Actions to test and deploy your code.

Windsurf can help you set up GitHub Actions configuration files.

Pro Tip: Use the command palette in Windsurf (Cmd/Ctrl+Shift+P) and type "Git" to see all available Git commands without leaving the editor.
