Netlify Guide

Netlify is a powerful platform for automating modern web projects, offering continuous deployment, serverless functions, and a global application delivery network. This guide will help you set up Netlify and deploy your first website.

What is Netlify?

Netlify is a web hosting infrastructure and automation technology company that offers hosting and serverless backend services for web applications and static websites. It provides continuous deployment from Git across a global application delivery network, serverless form handling, and more.

Creating a Netlify Account

Visit Netlify

Navigate to netlify.com in your web browser.

Choose a Sign-Up Method

Click the "Sign up" button and select from the available sign-up methods:

  • GitHub (recommended if you'll be deploying from GitHub)
  • GitLab
  • Bitbucket
  • Email

Using a Git provider simplifies deployment by giving Netlify access to your repositories.

Authorize Netlify

If you chose a Git provider, you'll be asked to authorize Netlify to access your account. This gives Netlify permission to read your repositories and set up webhooks.

Complete Your Profile

Fill out any required profile information and agree to Netlify's terms of service.

Netlify Plans

Netlify offers several pricing tiers:

  • Free: Perfect for personal projects and small websites
  • Pro: For professional websites with higher traffic needs
  • Business: For teams with multiple sites and advanced needs
  • Enterprise: For large organizations with custom requirements

The free tier includes:

  • Unlimited sites
  • Continuous deployment
  • HTTPS
  • Global CDN
  • 100GB bandwidth per month
  • Form handling (up to 100 submissions per month)
  • Community support

Deploying Your First Site

Go to the Netlify Dashboard

After signing in, you'll be taken to your Netlify dashboard. Click the "Add new site" button and select "Import an existing project".

Connect to Git Provider

Select your Git provider (GitHub, GitLab, or Bitbucket) where your project is hosted.

You'll see a list of your repositories. Select the one you want to deploy.

Repository Requirements

Ensure your repository contains the static website files (HTML, CSS, JS) or a static site generator project like React, Vue, Angular, etc.

Configure Build Settings

Netlify will automatically detect common project types and suggest build settings. You can review and adjust:

  • Branch to deploy: Usually main or master
  • Build command: The command to build your site (e.g., npm run build)
  • Publish directory: Where your built files are located (e.g., dist, build, or public)
Deploy the Site

Click "Deploy site" to start the deployment process. Netlify will clone your repository, run your build command, and deploy the result.

You can watch the deployment progress in real-time. When complete, Netlify will assign your site a random subdomain like random-name-123456.netlify.app.

Sample netlify.toml Configuration

For more control, you can add a netlify.toml file to your repository:

[build] publish = "dist" command = "npm run build" # Redirects and headers [[redirects]] from = "/*" to = "/index.html" status = 200 [[headers]] for = "/*" [headers.values] Cache-Control = "public, max-age=0, must-revalidate"

Key Netlify Features

Continuous Deployment

Netlify automatically builds and deploys your site when you push changes to your Git repository. Each commit triggers a new build.

Global CDN

Your site is distributed across a global content delivery network, ensuring fast loading times for visitors worldwide.

HTTPS by Default

Netlify provides free SSL certificates through Let's Encrypt, securing all sites with HTTPS automatically.

Deploy Previews

Each pull request gets its own preview deployment, making it easy to review changes before merging to your main branch.

Serverless Functions

Write serverless functions that run on-demand without having to manage servers. Perfect for APIs, form handling, and more.

Form Handling

Netlify can automatically handle form submissions from your site without requiring a backend server.

Deploy Rollbacks

Netlify keeps a history of all your deployments. If something goes wrong with a new deploy, you can instantly roll back to a previous version with one click.

Setting Up Custom Domains

Access Domain Settings

From your site dashboard, go to Site settings > Domain management.

Add Your Custom Domain

Click "Add domain" and enter your domain name (e.g., yourdomain.com).

You have two options:

  • Purchase a domain through Netlify - Convenient but potentially more expensive
  • Use a domain you already own - Requires updating DNS settings
Set Up DNS

If using your own domain, you'll need to configure your DNS settings:

  1. Option 1: Use Netlify DNS (recommended)
    • Netlify will guide you through updating your domain's nameservers at your registrar
    • This gives Netlify full control of DNS, simplifying setup
  2. Option 2: Keep your current DNS provider
    • Add CNAME records pointing to your Netlify site
    • Add A records for apex domains
Wait for DNS Propagation

DNS changes typically take 24-48 hours to fully propagate. Netlify will show the status of your domain setup.

Example DNS Records

If using your existing DNS provider, add these records:

# For a subdomain like www.yourdomain.com Type: CNAME Name: www Value: your-site-name.netlify.app # For an apex domain like yourdomain.com Type: A Name: @ Value: 75.2.60.5 # Alternative apex record Type: ALIAS or ANAME (if your provider supports it) Name: @ Value: your-site-name.netlify.app

HTTPS for Custom Domains

Netlify automatically provisions SSL certificates for your custom domains through Let's Encrypt. The process begins once your DNS is properly configured, and typically takes just a few minutes to complete.
