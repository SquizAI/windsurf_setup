Complete Workflow Guide

This guide explains the complete workflow for creating and deploying modern websites, from initial research to deployment. We'll cover how to use AI tools for research, create structured content, build interactive websites, and deploy them using GitHub and Netlify.

Step 1: Research with AI Tools (MCP)

Modern research leverages AI tools to gather information quickly and efficiently. Windsurf's Model Context Protocol (MCP) provides powerful research capabilities to gather information from various sources.

What is MCP?

The Model Context Protocol (MCP) is a system in Windsurf that connects AI systems with external tools and data sources. It allows Cascade to access specialized functions, external information, and services like web search and data analysis.

Define Your Research Questions

Start by clearly defining what information you need to gather. Break down broad topics into specific questions.

For example: "What are the key features of Windsurf?" or "How do I set up GitHub Pages?"

Research Using MCP

Use Windsurf's Cascade with MCP integrations to gather information:

  • Use the firecrawl MCP for web research and content extraction
  • Use context7 MCP for accessing library documentation
  • Request specific information from reliable sources
# Example request to Cascade Research the steps required to deploy a static website on Netlify from a GitHub repository. Please provide information on continuous deployment and custom domains.
Organize Research Findings

As Cascade provides information:

  • Identify key concepts and organize them into categories
  • Note important steps, tools, and terminology
  • Ask follow-up questions to clarify confusing points
  • Request specific examples for complex procedures
Research Best Practices
  • Ask specific questions rather than broad ones
  • Request up-to-date information when timeliness matters
  • Validate important technical information from multiple sources
  • Ask for step-by-step instructions for processes you'll need to follow

Step 2: Create Structured Markdown Content

Markdown provides an ideal format for structuring your content before converting it to a website. It's simple, readable, and can be easily converted to HTML.

Create a Markdown File

Create a file with the .md extension for your content. You can ask Cascade to generate this file based on your research.

# Example request to Cascade Based on our research, please create a well-structured markdown file that covers the complete guide for setting up GitHub and deploying to Netlify.
Structure Your Content

Organize your content with clear hierarchical structure:

# Main Title ## Section 1 Content for section 1... ### Subsection 1.1 More detailed content... ## Section 2 Content for section 2... - Bullet point 1 - Bullet point 2 1. Numbered step 1 2. Numbered step 2 > Important note or quote ![Image description](path/to/image.jpg) [Link text](https://example.com)
Preview Your Markdown

Use a markdown previewer (like the one in VS Code or online tools) to ensure your content is formatted correctly.

Check for proper heading levels, list formatting, and link functionality.

Benefits of Starting with Markdown

  • Focuses on content structure without design distractions
  • Easily readable and editable in any text editor
  • Can be converted to HTML automatically
  • Works well with version control systems like Git
  • Supported by many static site generators and CMS platforms

Step 3: Convert to HTML, CSS, and JavaScript

Once your content is structured in markdown, the next step is to convert it to a fully functional website with HTML, CSS, and JavaScript.

Convert Markdown to HTML

There are several approaches to convert your markdown to HTML:

  • Automatic conversion: Use tools like markdown-it, marked, or showdown
  • Static site generators: Jekyll, Hugo, or 11ty can convert markdown to a complete site
  • Manual conversion: Ask Cascade to convert your markdown to HTML with proper structure
// Example JavaScript to convert markdown to HTML const marked = require('marked'); const fs = require('fs'); // Read markdown file const markdown = fs.readFileSync('content.md', 'utf-8'); // Convert to HTML const html = marked.parse(markdown); // Save HTML fs.writeFileSync('output.html', html);

Step 4: Push Your Website to GitHub

Version control with GitHub helps track changes, collaborate with others, and simplifies deployment. This step covers how to upload your website to a GitHub repository.

Initialize a Git Repository

In your project directory, initialize a Git repository:

# Navigate to your project folder cd your-website-folder # Initialize a new git repository git init # Create a .gitignore file for unwanted files echo "node_modules/" > .gitignore echo ".DS_Store" >> .gitignore
Create a GitHub Repository

Create a new repository on GitHub (see the GitHub Guide section for details).

Copy the repository URL that GitHub provides.

Add Files and Commit

Add your files to Git and make your first commit:

# Add all files to staging git add . # Commit the files with a message git commit -m "Initial website commit"
Connect to GitHub and Push

Connect your local repository to GitHub and push your files:

# Add GitHub as a remote git remote add origin https://github.com/yourusername/your-repo-name.git # Push your files to GitHub git push -u origin main

If your main branch is called "master" instead of "main", use that name in the command.

GitHub Workflow for Updates

After the initial setup, follow this workflow for updates:

  1. Make changes to your files
  2. Test changes locally
  3. Add changes to staging: git add .
  4. Commit changes: git commit -m "Description of changes"
  5. Push to GitHub: git push

Step 5: Deploy to Netlify

Netlify provides fast, secure hosting with continuous deployment from GitHub. This step covers how to connect your GitHub repository to Netlify for automatic deployments.

Connect Netlify to GitHub

Log in to Netlify and click "Add new site" > "Import an existing project" > "GitHub".

Authorize Netlify to access your GitHub repositories if prompted.

Select Your Repository

Find and select the GitHub repository containing your website.

Configure Build Settings

Configure your build settings based on your project:

  • For a simple static site: Leave the build command blank and specify the directory containing your HTML files (e.g., ./ or ./dist)
  • For a site with a build process: Enter the build command (e.g., npm run build) and the output directory (e.g., build or dist)
Deploy Your Site

Click "Deploy site" and wait for the build process to complete.

Netlify will provide a random subdomain (e.g., random-name-123456.netlify.app) where your site is accessible.

Configure a Custom Domain (Optional)

To use a custom domain:

  1. Go to Site settings > Domain management
  2. Click "Add custom domain"
  3. Enter your domain name
  4. Follow Netlify's instructions to configure DNS settings
Continuous Deployment

With this setup, any changes you push to your GitHub repository will trigger an automatic deployment on Netlify. This enables a smooth workflow:

  1. Update your content locally
  2. Commit and push to GitHub
  3. Netlify automatically deploys the changes

Step 6: Ongoing Maintenance and Updates

A successful website requires ongoing maintenance and updates. This final step covers best practices for keeping your website up-to-date and functioning properly.

Monitor Your Site

Regularly check your site for issues:

  • Use Netlify's analytics to track visitor patterns
  • Check for broken links and images
  • Monitor site performance and load times
  • Review deployment logs for any build errors
Content Updates

Keep your content fresh and relevant:

  1. Update your markdown files with new content
  2. Convert to HTML if necessary
  3. Commit and push changes to GitHub
  4. Let Netlify automatically deploy the updates
Feature Enhancements

Improve your site over time:

  • Add new interactive elements
  • Improve accessibility
  • Enhance mobile responsiveness
  • Optimize performance

For significant changes, consider using git branches to develop and test before merging to your main branch.

The Iterative Process

Web development is an iterative process. As you learn more about your audience and their needs, you can refine your website accordingly. The workflow we've covered makes this iteration efficient:

  1. Research new topics or improvements with MCP
  2. Update your markdown content
  3. Enhance your website code
  4. Push changes to GitHub
  5. Netlify automatically deploys the updates
