Effective Prompting in Windsurf

What Makes Windsurf Different

Windsurf combines an AI coding assistant (Cascade) with powerful context awareness and tools. Unlike traditional AI assistants, Cascade can see your code, run commands, and use external tools through MCP. This requires adapting your prompting strategy for optimal results.

Prompting Best Practices

Follow these guidelines to get the most out of Windsurf:

  1. Be specific with your requests - Tell Cascade exactly what you want to accomplish
  2. One task at a time - Break complex tasks into smaller, manageable steps
  3. Provide context - Explain what you're working on and any relevant background
  4. Reference files explicitly - Mention file paths when working with existing code
  5. Request appropriate tools - Ask Cascade to use specific MCP servers when needed
Effective Prompts
  • "Research the latest best practices for responsive web design using the firecrawl MCP"
  • "Create a function that validates email addresses in JavaScript"
  • "Help me debug the error in src/components/Header.js"
Ineffective Prompts
  • "Make my website better" (too vague)
  • "Create a complete e-commerce platform" (too broad)
  • "Fix all the bugs" (lacks specific information)
Start with a Clear Goal

Begin your prompt by clearly stating what you want to accomplish. For example:

I want to create a responsive navigation menu for my website that collapses into a hamburger menu on mobile devices.
Provide Context and Constraints

Include relevant information about your project and any constraints:

I'm building a website using HTML, CSS, and vanilla JavaScript (no frameworks). The site needs to be compatible with IE11 and modern browsers.
Request Specific Research (When Needed)

Specify what you want to research and which tools to use:

Please use the firecrawl MCP to research current best practices for accessible navigation menus, focusing on WCAG 2.1 compliance.
Ask for Implementation

Request specific implementation help:

Based on this research, please help me implement a responsive navigation menu in my index.html, styles.css, and main.js files.

Complete Windsurf Workflow

In this section, we'll walk through the complete workflow we used to create the Windsurf Guide website, from initial research to deployment.

Markdown Content

Step 1: Research with MCP

We began by researching Windsurf, GitHub, and Netlify using the firecrawl MCP:

Please use the firecrawl MCP to research detailed information about Windsurf installation and setup. Include system requirements, installation process, and initial configuration steps.

This prompt triggered Cascade to use the firecrawl MCP server to search the web for up-to-date information about Windsurf.

Step 2: Create Structured Markdown Content

Based on the research, we created structured markdown content:

Based on our research, please create a well-structured markdown file for a comprehensive Windsurf Guide. Include sections for introduction, system requirements, installation process, initial setup, getting started with Cascade, and understanding MCP.

Cascade generated organized markdown content with proper headings, lists, and code blocks, making it easy to review and edit.

Step 3: Convert to HTML, CSS, and JavaScript

We converted the markdown to a component-based website:

Please convert the markdown content to a modern website structure with HTML, CSS, and JavaScript. Use a component-based approach where each section is in its own HTML file, loaded dynamically with JavaScript.

Cascade created a complete website structure with:

  • index.html as the main container
  • Component HTML files for each section
  • CSS for styling in a modern-styles.css file
  • JavaScript for dynamic loading and interactivity
Step 4: Push to GitHub

We created a GitHub repository and pushed our code:

# Initialize a Git repository git init # Add all files to staging git add . # Commit with an initial message git commit -m "Initial commit of Windsurf Guide website" # Add GitHub remote (replace with your repository URL) git remote add origin https://github.com/yourusername/windsurf-guide.git # Push to GitHub git push -u origin main
Step 5: Deploy to Netlify

Finally, we deployed the site to Netlify by connecting to our GitHub repository and configuring the deployment settings:

  1. Created a Netlify account and logged in
  2. Clicked "Add new site" > "Import an existing project"
  3. Connected to our GitHub repository
  4. Configured build settings (none needed for static site)
  5. Deployed the site
Key Benefits of This Workflow
  • AI-Powered Research: Leveraging MCP for comprehensive, up-to-date information
  • Structured Content: Markdown provides a clean, organized format for content
  • Modern Web Development: Component-based approach for maintainability
  • Version Control: GitHub for tracking changes and collaboration
  • Automated Deployment: Netlify for continuous deployment from GitHub

Windsurf Prompt Examples by Task

Below are example prompts for various tasks in the web development workflow with Windsurf. These demonstrate best practices for prompting Cascade to get optimal results.

Research Prompts

# Basic research prompt Use the firecrawl MCP to research the latest best practices for responsive web design in 2025. Focus on mobile-first approaches, accessibility, and performance optimization. # Targeted research prompt Use the firecrawl MCP to find detailed information about GitHub Pages deployment options. I need to understand the difference between project pages and user pages, custom domain configuration, and build options. # Comparative research prompt Use the firecrawl MCP to compare Netlify, Vercel, and GitHub Pages as hosting options for a static React website. Focus on build times, deployment features, and free tier limitations.

Content Creation Prompts

# Markdown content creation Based on our research about Netlify, please create a structured markdown file that explains the process of deploying a website to Netlify from GitHub. Include sections for account setup, repository connection, build configuration, and custom domain setup. # Structured guide creation Create a comprehensive markdown guide for GitHub with the following sections: 1. What is GitHub (brief introduction) 2. Creating an account 3. Setting up your first repository 4. Basic Git commands 5. Collaborative workflow 6. GitHub Pages deployment # Technical documentation Create markdown documentation for a JavaScript function that handles form validation. The function should validate email, password strength, and form completeness. Include code examples, parameter explanations, and usage examples.

Web Development Prompts

# HTML structure creation Create an HTML structure for a modern website with the following sections: header with navigation, hero section, features section with cards, about section, pricing tables, testimonials, contact form, and footer. Use semantic HTML5 elements. # CSS styling prompt Create a CSS file with modern styling for our website. Use CSS variables for a consistent color scheme, implement responsive design with media queries, and add subtle animations for interactive elements. The design should be clean and professional with a color scheme based on blue (#2963ff) as the primary color. # JavaScript functionality Create a JavaScript file that implements the following functionality for our website: 1. Smooth scrolling for navigation links 2. Toggle mobile menu for small screens 3. Form validation for the contact form 4. Dark mode toggle with local storage persistence 5. Lazy loading for images

Component Development Prompts

# Header component Create a header component in HTML with a logo, navigation menu, and call-to-action button. The navigation should collapse to a hamburger menu on mobile devices. Include the necessary CSS and JavaScript. # Tabbed interface component Create a tabbed interface component that allows users to switch between different content sections. The component should have clickable tabs at the top, and only show the content of the active tab. Include HTML, CSS, and JavaScript for this component. # Modal dialog component Create a modal dialog component that can be triggered by clicking a button. The modal should appear with an animation, overlay the rest of the page, and include a close button. Include HTML, CSS, and JavaScript for this component.

Deployment Prompts

# GitHub workflow prompt Help me push my website code to GitHub. I need to initialize a repository, add all files, create an initial commit, and push to a new repository. My website files are in the current directory. # Netlify deployment prompt Guide me through deploying my website to Netlify from my GitHub repository. I want to connect Netlify to my GitHub repository at username/website-repo, configure it to deploy the main branch, and set up a custom domain. # Continuous integration prompt Help me set up GitHub Actions for continuous integration with my website project. I want to run linting and test scripts whenever code is pushed to the repository, and only allow PRs to be merged if all checks pass.

Combining Prompts for Complex Workflows

For complex tasks, you can combine multiple prompts in sequence to guide Windsurf through the full workflow:

  1. Start with research: "Use firecrawl MCP to research..."
  2. Follow with content creation: "Based on the research, create a markdown file..."
  3. Request web development: "Convert the markdown to HTML, CSS, and JS..."
  4. Finally, deployment: "Help me push this code to GitHub and deploy to Netlify..."

This step-by-step approach keeps tasks manageable and allows you to review and provide feedback at each stage.
