GitHub Guide

GitHub is a web-based platform that uses Git for version control, allowing developers to store, track, and collaborate on code projects. This guide will walk you through setting up a GitHub account and creating your first repository.

What is GitHub?

GitHub is where over 100 million developers shape the future of software, including storing code, managing projects, and collaborating with teams. It provides powerful tools for version control, issue tracking, and continuous integration.

Creating a GitHub Account

Visit the GitHub Website

Navigate to github.com in your web browser.

Fill Out the Sign-Up Form

On the homepage, you'll see a sign-up form. Enter your:

  • Email address
  • Choose a strong password
  • Select a username (this will be your GitHub identity)
Verify Your Account

Complete the verification process (may include solving a puzzle).

Check your email to verify your email address by clicking the link GitHub sends you.

Personalize Your Experience

GitHub will ask you some questions about your experience level and interests to help tailor your experience. You can also skip this step.

Pro Tips for Your GitHub Account

  • Choose a professional username as it will be visible in your contributions
  • Set up two-factor authentication for enhanced security
  • Add a profile picture and bio to make your profile more personalized

Creating Your First Repository

Navigate to Repositories

Once logged in, click the "+" icon in the upper right corner and select "New repository".

Configure Your Repository

Fill out the repository creation form:

  • Repository name: Choose a clear, descriptive name (e.g., "windsurf-guide")
  • Description: Add a short description of your project
  • Visibility: Choose Public or Private
  • Initialize: Select "Add a README file" to start with basic documentation
Create Repository

Click "Create repository" to finish. You'll be taken to your new repository page.

Public vs. Private Repositories

Public repositories are visible to anyone on the internet, while private repositories are only accessible to you and people you explicitly share them with. Free GitHub accounts have unlimited public repositories and limited private repositories.

Basic GitHub Operations

Common Git Commands

These commands will help you manage your repository from your local machine:

# Clone a repository to your local machine git clone https://github.com/yourusername/your-repo.git # Check the status of your repository git status # Add files to staging area git add . # Add all files git add filename.ext # Add specific file # Commit your changes git commit -m "Your commit message" # Push changes to GitHub git push origin main # Pull latest changes from GitHub git pull origin main # Create and switch to a new branch git checkout -b new-branch-name

Understanding Branches

Branches allow you to develop features, fix bugs, or experiment with new ideas in a contained area of your repository, without affecting the main project. The default branch is usually called "main" (or "master" in older repositories).

Pull Requests

Pull requests let you tell others about changes you've pushed to a branch in a repository. Once a pull request is opened, you can discuss and review the potential changes with collaborators before your changes are merged into the main branch.

Hosting with GitHub Pages

GitHub Pages is a free hosting service that lets you publish static websites directly from your GitHub repository.

Enable GitHub Pages

From your repository, go to Settings > Pages.

Choose a Source

Select the branch you want to publish (usually "main").

Select the folder (usually "/" or "/docs") that contains your website files.

Save and Access Your Site

Click "Save". After a few minutes, your site will be published at https://yourusername.github.io/repository-name/

GitHub Pages Limitations

  • Only static content (HTML, CSS, JavaScript) is supported
  • Repositories have a recommended limit of 1GB
  • Published sites have a soft bandwidth limit of 100GB per month
  • Site build process is limited to 10 builds per hour
